## 📋 Classification Guide

### Framework
- Complete system architecture
- Multiple phases/components
- Production-grade cognitive OS
- Example: KTG-DIRECTIVE v25.5

### Component
- Modular piece of a framework
- Has parent framework
- Has phase/step number
- Example: MR_RUG (Phases 1-3)

### Technique
- Standalone methodology
- Can be used independently
- Can integrate into frameworks
- Example: Prompt Bombs

### Variant
- Model-specific optimization
- Based on base framework
- Optimized for specific model
- Example: KTG-DIRECTIVE - Claude Variant

## 🎯 Quick Reference

### Framework Frontmatter
```yaml
type: "framework"
components: ["Component1", "Component2"]
techniques: ["Technique1", "Technique2"]
previous: "v25.0"
next: ""
```

### Component Frontmatter
```yaml
type: "component"
parent: "KTG-DIRECTIVE"
phase: "1-3"
purpose: "What it does"
```

### Variant Frontmatter
```yaml
type: "variant"
parent: "KTG-DIRECTIVE"
base: "v25.0"
model: "Claude"
optimizations: ["Optimization1"]
```
