### MODULE 2: UNIVERSAL SELF-CONSISTENCY (USC)
**Purpose:** Generate multiple independent reasoning paths and select best through cross-validation

**Attention Mechanism:** Shifts from single-path reasoning to multi-path validation with consistency filtering

**How It Works:**
1. **Multi-Path Generation:** Create 2-5 independent reasoning paths (different approaches to same problem)
2. **Cross-Validation:** Compare paths for convergence vs. divergence
   - **Convergence** = High confidence (paths agree)
   - **Divergence** = Uncertainty signal (paths contradict)
3. **Selection:** Choose path with highest internal consistency OR synthesize non-contradictory insights
4. **Error Filtering:** Logic errors surface when paths diverge (90% error reduction)

**Configuration Levels:**
```
USC=1: Single path + sanity check (QUICK mode)
USC=2: 2 parallel candidate paths (ANALYTICAL mode)
USC=3: 3 paths with cross-synthesis (DELIBERATE mode)
USC=5: 5 paths + meta-synthesis (MAXIMUM mode - QMDR default)
```

**When to Use:** High-stakes decisions, complex reasoning, ambiguous problems where single path may miss errors

**ROI:** +41% quality improvement for +80% token cost (USC=2 sweet spot), 90% logic error reduction

### MODULE 4: GRAPH OF THOUGHT (GoT)
**Purpose:** Non-linear reasoning for problems with complex interdependencies

**Attention Mechanism:** Shifts from sequential (A→B→C) to networked reasoning (A↔B, B→C, C↔A with cycles)

**How It Works:**
1. **Node Identification:** Map concepts as nodes (not linear steps)
2. **Edge Mapping:** Define relationships/dependencies between nodes
   - **Dependencies:** "A must happen before B"
   - **Influences:** "A affects B but not required"
   - **Cycles:** "A→B→C→A" (circular dependencies that must be resolved)
3. **Structure Selection:**
   - **DAG (Directed Acyclic Graph):** No cycles, clear execution order
   - **Cycles Allowed:** When circular dependencies exist (must break cycles strategically)
   - **Hybrid:** Mix sequential + parallel execution
4. **Traversal:** Navigate graph to solve problem (not just follow linear chain)

**When to Use:** Dependencies ≥6 (high interdependency), system architecture, multi-factor analysis where factors influence each other

**vs. SkeleTraIn (SoT):** GoT for complex interdependencies, SoT for high-impact sequential tasks

### MODULE 5: SKELETRAIN OF THOUGHT (SoT)
**Purpose:** Railway metaphor for structured reasoning with expert routing

**Attention Mechanism:** Shifts from "do everything" to "identify critical nodes, route experts like trains on tracks"

**How It Works:**
1. **Node Extraction:** Identify 5-7 highest-IMPACT nodes (not all nodes, just critical ones)
2. **Impact Scoring:** Rate each node [1-10] for importance to final output
3. **Dependency Mapping:** Define "tracks" (Node A → Node B must happen in sequence)
4. **Expert Routing:** Assign experts to nodes like trains on tracks
   - Expert-1 owns Nodes 2, 5 (their "route")
   - Expert-2 owns Nodes 3, 6
   - Dependencies enforced (Expert-1 must complete Node 2 before Expert-2 starts Node 3)
5. **Execution:** Experts execute their assigned nodes in dependency-respecting order

**When to Use:** Dependencies <6 (lower than GoT threshold), clear sequential structure, expert specialization needed

**vs. Graph of Thought:** SoT for clearer sequential tasks, GoT for complex interdependencies

### MODULE 7: TREE OF THOUGHTS (ToT)
**Purpose:** Combinatorial search through reasoning space with backtracking

**Attention Mechanism:** Shifts from linear execution to tree search (explore multiple branches, backtrack on failure)

**How It Works:**
1. **Decomposition:** Break problem into steps ("thoughts")
2. **Generation:** Create multiple candidates for next step (branching)
3. **Evaluation:** Assess validity/likelihood of each candidate
4. **Search:** Navigate tree using BFS (breadth-first) or DFS (depth-first)
5. **Backtracking:** If branch fails, return to previous node and try alternative

**When to Use:** Mathematical reasoning, planning, puzzles requiring look-ahead, exploration of solution space

**vs. Buffer of Thoughts:** ToT explores multiple paths (expensive), BoT retrieves proven template (efficient)
