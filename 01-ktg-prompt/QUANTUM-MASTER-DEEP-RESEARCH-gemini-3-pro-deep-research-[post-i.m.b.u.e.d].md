2. PHASE 1: THE INVERSION (FAILURE MODES)
We invert the problem: "Why will this stack fail?"
The shift to Gemini 3 Pro introduces specific new risks regarding latency and state management.
Risk Domain
Specific Failure Mode
The Mitigation Strategy
Vercel Limits
The "Thinking" Timeout. Gemini 3 Pro's "High Thinking" mode can take >60s to reason. Vercel Pro functions time out at 60s.
Streaming + Async Handoff. Use streamUI to send a "Thinking..." component immediately. For video ingestion, offload to n8n (async), do not block the UI.
State Drift
Lost "Thought Signatures". If we don't pass the encrypted thought_signature back to Gemini in Turn 2, it lobotomizes the agent (it forgets why it recommended X).
Strict State Management. The Vercel AI SDK useAIState must persist the thought_signature object, not just the text history.
n8n / YouTube
Quota Explosion. The YouTube API has a 10k daily quota. A single "Get Transcripts" loop for 50 videos can burn 5,000 units instantly.
The "Grey Zone" Scraper. Use n8n-nodes-webpage-content-extractor or a scraping API (e.g., Scrape.do) to bypass the official API for captions.
Database
Write/Read Lock. Heavy scraping (Write) blocks User Search (Read) on the HNSW Vector Index.
Unlogged Staging Tables. Ingest data into a temporary table without WAL logging, then batch-merge to the main vector table during Perth off-hours (3 AM AWST).

3. PHASE 2: THE FEASIBILITY SPEC (TECHNICAL DEEP DIVE)
A. The "Perth Utility Score" Algorithm
Gemini 3 Pro calculates this score (0-10) during ingestion. It is not a generic review; it is a compatibility filter.
Scoring Criteria (Prompt Logic):
Timezone Sovereign: Does the UI support Australia/Perth (UTC+8) specifically, or does it force "AEST" (Sydney)? (+2 points)
AUD Billing: Can you pay in AUD to avoid FX fees? (+1 point)
Support Window: Is support available 09:00-17:00 AWST? (Most US/East Coast support opens at 6 AM or 4 PM Perth time). (+3 points)
Compliance: Does it handle WA Statutory LSL (8.667 weeks after 10 years)? (+2 points)
Solo-Viability: Is the "Time to First Value" under 2 hours for a non-technical owner? (+2 points)
