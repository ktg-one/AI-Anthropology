# X Thread Template

## Structure

```
HOOK (Tweet 1):
- Pattern interrupt or bold claim
- 50-80 chars max
- No hashtags

BODY (Tweets 2-5):
- One insight per tweet
- Use numbers/specifics
- Thread emoji: 🧵

CTA (Final Tweet):
- Clear action
- Link if relevant
- Soft ask
```

## Example: Technique Post

```
Tweet 1 (Hook):
I've been prompt engineering for 3 years.

Here's what nobody tells you about context compression: 🧵

Tweet 2:
Most people think summarization = compression.

Wrong.

Real compression preserves RETRIEVABILITY, not just information.

Tweet 3:
The secret? Progressive Density Layering (PDL):

• L1: Knowledge (40%) - facts, decisions
• L2: Relational (25%) - dependencies  
• L3: Contextual (20%) - patterns
• L4: Meta (15%) - session fingerprint

Tweet 4:
Result: 6:1 compression ratio
Forensic recall: 9.52/10

Tested across Claude, GPT-4, Gemini, Grok.

Tweet 5 (CTA):
Full framework + template at ktg.one

Questions? Drop them below.
```

## Hashtag Strategy

- Only on final tweet
- Max 2 hashtags
- Use: #PromptEngineering #AI

## Timing

- Best: 9am, 12pm, 5pm (target timezone)
- Avoid: weekends for technical content
