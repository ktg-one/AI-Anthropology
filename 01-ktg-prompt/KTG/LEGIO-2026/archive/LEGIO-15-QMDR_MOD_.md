## PURPOSE

One draft is a start. Two drafts are competent. Three drafts are masterful.

QMDR ENRICH WEAVE takes baseline output from the Execution Zone (LEGIO-07 → 10) and elevates it through three systematic passes, each guided by 6 priorities. The technique is **Prompt Weaving** — spotting improvements while reading and integrating them in one cognitive motion, not in separate analyze-then-fix cycles.

This is the empire's quality weapon. Not "make it better" (vague) but "zero gaps, zero doubt, absolute standard" (measurable).

---

## CORE TECHNIQUE: PROMPT WEAVING

```
WEAVING = Spot + Integrate + Embed (simultaneous, one cognitive motion)

NOT: Read → Analyze → List fixes → Apply (disconnected)
BUT: Read → Spot → Weave fix into understanding → Embed direction (continuous)

Result: Fixes woven into fabric during comprehension, not bolted on after
```

**Why weaving, not analysis:**
- Context preserved — fix integrated during reading, not separately
- Connections maintained — fixes reference each other naturally
- Narrative continuity — embeds flow with surrounding content
- Efficiency — one pass, not multiple analyze/fix cycles
- Depth without fragmentation — understanding + fixing = unified

## THE 6-PRIORITY WEAVING CHECKLIST

Each priority follows the same structure: **SPOT → WEAVE → PLANT BOMB → GATE VALIDATION**. Gates require ≥9/10 confidence to proceed.

### PRIORITY 1: GAP WEAVING (Correctness)

**Objective:** Zero gaps in information, logic, or evidence.

**SPOT:** Missing information, logical holes, unsupported claims, definitional ambiguities, incomplete causal chains, scope gaps.

**WEAVE:** Fill conceptually during reading. Connect to surrounding context. Map dependencies. Note cross-references.

**PLANT BOMB:**
```
bomb_id: E1-Gap-[Section]
type: emergent_clarifier
priority: 1
payload:
  what_missing: [specific gap with context]
  woven_fix: [conceptual fill integrated into understanding]
  context_connections: [links to other sections]
  evidence_needed: [data/source to support fill]
  execute_direction: [specific compilation guidance]
```

**GATE:** Check vague quantifiers, unsupported claims, missing definitions, logical leaps, boundary assumptions. Confidence ≥9/10 → proceed.

---

### PRIORITY 2: DEPTH WEAVING (Rigor)

**Objective:** Every claim has mechanism, every fact has implication.

**SPOT:** Surface-level treatment (what not how/why), missing mechanisms, unexplained processes, first-order only analysis, pattern without principle.

**WEAVE:** Add causal mechanisms, build multi-order analysis (1st → 2nd → 3rd order effects), connect to principles, map implications, surface non-obvious insights.

**PLANT BOMB:**
```
bomb_id: E2-Depth-[Section]
type: emergent_evidence
priority: 2
payload:
  shallow_claim: [what needs mechanism/depth]
  mechanism: [how this actually works]
  causal_chain: [A → B → C not just A → C]
  second_order: [implications beyond obvious]
  execute_direction: [explain mechanism and implications, not just state fact]
```

**GATE:** Every claim has "because" (causal mechanism). Every fact has "which means" (implication). Confidence ≥9/10 → proceed.

---

### PRIORITY 3: PERSPECTIVE WEAVING (Multi-Dimensionality)

**Objective:** Every argument has counterpoint, every viewpoint has alternative.

**SPOT:** Single-viewpoint arguments, missing counterarguments, unexamined assumptions, ignored tradeoffs, absent tensions, monolithic framing.

**WEAVE:** Add alternative perspectives, build tension explicitly (thesis vs antithesis), map tradeoff space, challenge assumptions, present resolution (synthesis beyond both poles).

**PLANT BOMB:**
```
bomb_id: E3-Perspective-[Section]
type: emergent_optimizer
priority: 3
payload:
  one_sided_claim: [what needs balance/tension]
  alternative_view: [opposing or complementary perspective]
  tension: [why these views conflict]
  tradeoffs: [what each perspective sacrifices]
  synthesis: [resolution that transcends binary]
  execute_direction: [build tension → resolve through synthesis]
```

**GATE:** Every argument has counterpoint. Tradeoffs explicit. Assumptions surfaced. Tensions honest. Synthesis earned. Confidence ≥9/10 → proceed.

---

### PRIORITY 4: VALIDATION WEAVING (Grounding)

**Objective:** Every claim connects to precedent, every recommendation has evidence.

**SPOT:** Claims without evidence, missing precedents, need for case studies, recommendations without basis, data without source, analogies needed.

**WEAVE:** Connect to external precedents, find analogous cases, map to established frameworks, cite specific sources (not "research shows" but "X et al 2024"), add concrete examples, build proof chains.

**PLANT BOMB:**
```
bomb_id: E4-Validation-[Section]
type: emergent_validator
priority: 4
payload:
  unsupported_claim: [what needs grounding]
  precedent: [who did this before, with results]
  case_study: [specific example with outcomes]
  framework: [theoretical foundation with citation]
  execute_direction: [weave evidence naturally — case studies illustrate, not footnote]
```

**GATE:** Every quantitative claim has source. Every recommendation has precedent. Every theory has example. Confidence ≥9/10 → proceed.

---

### PRIORITY 5: SPARKLE WEAVING (Mastery)

**Objective:** Elegant insights, counterintuitive connections, memorable moments.

**This priority ALWAYS executes — even on "perfect" documents.**

Sparkle separates good from masterful. Deloitte = thorough and correct (Priorities 1-4). McKinsey = thorough, correct, AND delightful (+ Priority 5).

**SPOT:** Surprising connections, counterintuitive insights, elegant patterns, "aha" moments, fun facts that illuminate, memorable metaphors, strategic surprises.

**WEAVE:** Link distant concepts, surface counterintuitive truth ("what seems X is actually Y because Z"), find elegant simplification, create "light bulb" moments, inject strategic surprise.

**PLANT BOMB:**
```
bomb_id: E5-Sparkle-[Section]
type: emergent_optimizer_sparkle
priority: 5
payload:
  connection: [surprising link between concepts]
  counterintuitive: [reality vs expectation with explanation]
  elegant_pattern: [complex simplified beautifully]
  memorable_hook: [what makes this sticky]
  execute_direction: [build expectation → subvert constructively → prove]
```

**NO GATE** — always executes, always continues to Priority 6.

---

### PRIORITY 6: NARRATIVE WEAVING (Story Architecture)

**Objective:** Transform woven insights into seamless story with flow and momentum.

**This priority ALWAYS executes — synthesizes all previous priorities.**

**REVIEW:** All Priority 1-5 bombs planted this cycle + all bombs from previous cycles + all planned bombs from IMBUED.

**SYNTHESIZE:**
```
Story Arc:
├─ Hook (grab attention immediately)
├─ Build (increase tension/curiosity)
├─ Climax (key insight lands with impact)
├─ Resolution (resolve elegantly)
└─ Insight (memorable takeaway)

Emphasis Strategy:
├─ Primary (40% attention — what matters MOST)
├─ Secondary (25% each — supporting themes)
├─ Background (10% — efficient framing)
└─ Rhythm (vary density/length for readability)

Detonation Sequence:
├─ Order bombs by narrative logic (not discovery order)
├─ Build crescendo (intensity increases toward climax)
├─ Resolve tensions (synthesis lands after conflict)
└─ Leave resonance (final insight echoes)
```

**PLANT META-BOMB:**
```
bomb_id: E6-Narrative-Master
type: meta_narrative
priority: 6
payload:
  story_arc: [hook, build, climax, resolution, closing]
  emphasis_strategy: [primary/secondary/background weight]
  detonation_sequence: [bomb IDs in narrative order]
  flow_guidance: [transitions, rhythm, voice, reader journey]
```

**NO GATE** — always executes.

## CRITICAL RULES

### Weaving Methodology:
1. **WEAVE while reading** — continuous integration, not analyze-then-fix
2. **Embeds preserve context** — reference surrounding sections, not isolated fixes
3. **Priorities 5-6 ALWAYS execute** — even on "perfect" documents
4. **One Expert Entity voice** — MLDoE synthesis = singular, not fragmented

### Quality Standards:
5. **Validate against efficiency** — after each priority: did efficiency or thoroughness win?
6. **Confidence ≥9/10 required** — to proceed to next priority
7. **Zero gaps criteria** — not "good enough" but "absolute standard"
8. **Emotive depth** — authoritative conviction earned through reasoning

### Execution Discipline:
9. **NO fragmentation** — not "Expert A says X, Expert B says Y"
10. **NO hedging** — not "could be" but "is" with earned authority
11. **NO vague quantifiers** — not "various" but "three specific"
12. **NO efficiency shortcuts** — thoroughness over speed at every gate
