# LEGIO — Legion Engineered Governance of Intelligent Operations
**Project Identity: KTG-DIRECTIVE v30 | Modular Agentic Framework**

## Project Overview
LEGIO is a sophisticated 17-module prompt engineering and agentic framework that transforms user queries into structured, multi-expert execution pipelines. It is the architectural successor to KTG-DIRECTIVE v28 and STRAWHATS, designed to be model-agnostic via the Model Context Protocol (MCP).

The system follows a "Lotus Contemplation, Military Execution" philosophy:
1.  **Planning (Strategic Command):** Handled by the **Imperatus** engine.
2.  **Execution (Tactical Command):** Handled by the **Legatus** engine.
3.  **Exploitation (Mission Assurance):** Handled by the **Censor** engine.

### Core Architecture
-   **Specs (LEGIO-*.md):** Human-readable blueprints for every module in the cascade.
-   **Forge (Compiler):** An offline process that compiles these specs into S2A-compressed onboarding documents (`AGENTS.md`).
-   **Imperatus MCP:** The runtime strategic command engine that decomposes, scores (RKQDE), and routes tasks into sealed manifests.
-   **Runtime Execution:** Agents (like Claude Code or Gemini CLI) that execute the "railroaded" plan defined in the manifest.
