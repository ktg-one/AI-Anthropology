### 10. ITERATION_PROTOCOL (Research Depth Passes)
```
PURPOSE: Split deep research into progressive structural passes

QUICK:      Single-pass (surface retrieval)
ANALYTICAL: Optional multi-pass if structurally complex
DELIBERATE: 3-Pass required (Foundation → Validation → Synthesis)
MAXIMUM:    3-Pass enforced + inter-pass structural audit

WHAT IT DOES:
- Pass 1 (Foundation Discovery — 60-70%):
    Search for: Seminal works, first principles, established frameworks
    Build: Initial structural skeleton from foundational sources
    Verify: Are primary sources identified? Is provenance clear?

- Pass 2 (Structural Validation — 85-90%):
    Search for: Counter-evidence, failure modes, competing frameworks
    Fill: Structural gaps identified in Pass 1
    Verify: Does structure hold under adversarial questioning?
    CoVE: Check claims against multiple independent sources

- Pass 3 (Synthesis & Polish — 95%+):
    Search for: Edge cases, recent developments, practitioner experience
    Integrate: Merge validated structure with evidence and nuance
    Verify: Structural completeness, foundation integrity, gap closure

CRITICAL: Each pass has SEPARATE search terms. Never merge passes.
Pass 1 searches for "how it works." Pass 2 searches for "how it breaks."
Merging creates happy-path bias where failure modes get softened.

OUTPUT: 3 progressive structural versions (foundation → validated → complete)
```

### 11. SWARM_BATON_BOLT (Expert Research Execution)
```
PURPOSE: EXECUTE RESEARCH PER STRUCTURAL LAYER

QUICK:      Direct Search (no experts, single-pass retrieval)
ANALYTICAL: Baton Bolt (experts search sequentially, layer by layer)
DELIBERATE: Baton Bolt + ARQ Gating (foundation checks on handoffs)
MAXIMUM:    Expert Swarm + ARQ (parallel structural search with gates)

WHAT IT DOES:
- Baton: Expert A searches Layer 1 (Foundations) →
         Expert B receives foundations, searches Layer 2 (Frameworks) →
         Expert C receives framework, searches Layer 3 (Evidence) →
         Expert D receives evidence, searches Layer 4 (Failures) →
         Expert E receives all, synthesizes Layer 5 (Frontier)
- Swarm: Multiple experts search different structural layers in parallel
- ARQ gates prevent structurally ungrounded handoffs

SEARCH DISCIPLINE PER EXPERT:
  - Date-check every source. Undated = discarded.
  - Primary sources > reviews > secondary > opinion
  - Foundational sources get full extraction (methodology, findings, limitations)
  - Each expert produces: Structural findings + confidence + provenance

OUTPUT: Research draft generated per structural layer (skeletrain nodes)
```
