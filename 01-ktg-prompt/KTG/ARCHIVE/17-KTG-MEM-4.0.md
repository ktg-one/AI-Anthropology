## PDL THEORETICAL FOUNDATION

### Progressive Density Layering vs Summarization

**Summarization asks:** "What are the key points?" (optimized for human reading)  
**PDL asks:** "What must be preserved for fresh model instance to continue this work?" (optimized for machine restoration)

### The Four-Layer Density Hierarchy

**Layer 1: KNOWLEDGE** - Core facts and entities (traditional CoD target)  
**Layer 2: RELATIONAL** - Connections between concepts, cross-domain links  
**Layer 3: CONTEXTUAL** - Domain-specific constraints, goals, user preferences  
**Layer 4: META-COGNITIVE** - Reasoning patterns, decision history, WHY choices made

**Standard summarization captures Layer 1 only. PDL explicitly preserves Layers 2-4, critical for context continuation.**

### The 0.15 Crystallization Point

From PDL research: `if density(P_i) ≥ 0.15 entities/token then break`

**This is the OPTIMAL compression point:**
- Maximum density WITH zero information loss
- Not a "danger threshold" - it's the TARGET
- Optimized for machine recall, not human reading
- Cross-domain relationships preserved at this density
