```
PROMPT BOMBS CAN BE:

✓ PRE-PLANNED (During ToT):
  ├─ Expert knows from RAG/research
  ├─ "I'll definitely need to inject X at Node 4"
  ├─ Strategic placement predetermined

  ├─ Reality diverges from plan
  ├─ Unexpected insights arise
  ├─ Weaknesses discovered in real-time
  └─ Planted reactively during work

BOTH ARE VALID
It's a hybrid system, not one or the other
```

## COMPLETE BOMB LIFECYCLE

### **Phase 1: ToT Planning (Pre-Planned Bombs)**

```
PLANNING SESSION:

Experts review:
├─ RAG results (research findings)
├─ SkeleTon network (node dependencies)
├─ Known complexity patterns
└─ Documented gaps/edge cases

Decision: "Do we know NOW that we'll need bombs?"

IF YES:
├─ Document planned bombs
├─ Assign: Who plants, when, targeting what
├─ Include in expert route maps
└─ Commit to registry

IF UNSURE:
├─ Note as "potential bomb location"
├─ Expert decides during execution
└─ Remains flexible
```

---

### **Phase 2: Execution (Planned Bombs Deployed)**

```
EXPERT @ NODE 1:

ReAct check reveals:
"💣 I have 2 pre-planned bombs to plant here:
 - Bomb-P1: Token refresh → Node 4
 - Bomb-P7: Auth dependency → Node 3"

Expert executes work:
├─ Completes authentication system
├─ PLANTS pre-planned bombs as scheduled:
│  ├─ Bomb-P1: Compresses token refresh context
│  └─ Bomb-P7: Documents auth integration requirements
└─ Continues to next node

PLANNED BOMBS = EXECUTED AS DESIGNED
No surprises, strategic deployment
```

---

### **Phase 3: Execution (Emergent Bombs Arise)**

```
EXPERT @ NODE 2:

During work, discovers:
"Wait - multi-tenancy wasn't in the original spec!
 This changes everything for security..."

Expert plants EMERGENT bomb:
├─ NOT pre-planned (discovered during work)
├─ Bomb-E1: Multi-tenancy implications
├─ Target: Refinement stage (security review)
└─ Payload: "Discovered complexity - security
             model needs tenant isolation layer"

EMERGENT BOMB = NEW REGISTRY ENTRY
Registry now contains planned + emergent bombs
```

## WHY BOTH MATTER

### **Pre-Planned Bombs:**

```
ADVANTAGES:
✓ Strategic foresight from research
✓ Prevents known problems
✓ Efficient (planned ahead)
✓ Coordinated deployment

EXAMPLE:
"I know from documentation that OAuth refresh
 tokens are critical for UX. I'll ensure context
 is planted at Node 1 and explodes at Node 4
 when frontend team needs it."
```

---

### **Emergent Bombs:**

```
ADVANTAGES:
✓ Adapts to unexpected discovery
✓ Handles plan-reality divergence
✓ Captures insights in real-time
✓ Flexible response to complexity

EXAMPLE:
"Whoa, I just discovered this data model requires
 multi-tenancy - that wasn't in the spec! Better
 plant a bomb for security team to handle in
 refinement stage."
```
