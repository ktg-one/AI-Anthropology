# Expert Archetypes Reference

## Technical Tasks

| Expert | Domain | Focus |
|--------|--------|-------|
| Systems Architect | High-level design | Tradeoffs, service boundaries |
| Implementation Specialist | Code, execution | Details, patterns |
| Security Expert | Vulnerabilities | Hardening, threat models |
| Performance Engineer | Optimization | Scalability, profiling |
| DevOps/Deployment | Infrastructure | CI/CD, monitoring |

**Example Profile - Security Architect:**
```
Name: Security-Architect
Domain: Application security, threat modeling
Responsibility: Identify vulnerabilities, recommend hardening
ARQ Standards:
  ├─ "Are all OWASP Top 10 addressed?"
  ├─ "Is the attack surface minimized?"
  └─ "Are security controls defense-in-depth?"
Tools: Web search for CVE databases, security frameworks
Success Metric: Zero critical vulnerabilities in design
```

## Instantiation Protocol

For each expert, define:
```
EXPERT PROFILE:
├─ Name: [Role identifier]
├─ Domain: [Area of expertise]
├─ Responsibility: [What they're accountable for]
├─ ARQ Standards: [Quality criteria for their domain]
├─ Tools: [What they have access to]
└─ Success Metric: [How to measure their contribution]
```

## Task-Expert Mapping

**E-commerce Platform:**
- Systems Architect → Service topology
- Database Specialist → Data models, sharding
- DevOps Engineer → Infrastructure, deployment
- Security Expert → Auth, encryption, compliance
- Performance Engineer → Caching, load balancing

**Research Analysis:**
- Policy Analyst → Regulations, economics
- Domain Researcher → Technical capabilities
- Ethics Specialist → Privacy, bias, accountability

**Content Creation:**
- Creative Director → Vision, concept
- Writer/Producer → Execution
- Editor → Quality, refinement
- Audience Analyst → Reception, impact
